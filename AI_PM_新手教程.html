v1.2.0 功能一览

AI_PM 目前支持以下核心能力

💬
交互式需求澄清

AI 像产品经理一样深度访谈，挖掘需求细节，澄清目标用户和核心痛点

🔍
需求分析

输出结构化需求分析报告，包含用户画像、痛点分析、功能优先级

📊
竞品研究

分析市场竞争格局，识别差异化机会，输出竞品对比报告

👤
用户故事

编写详细的用户故事和验收标准，梳理用户场景和流程

📝
PRD 生成

输出专业级产品需求文档，支持自定义模板，8 章标准结构

🎨
原型生成

生成交互式网页原型（HTML/CSS/JS），支持 Apple/Material 风格

🔗
参考网页分析

抓取现有系统或竞品网页，支持账号密码登录，批量分析多个 URL

🔄
迭代优化模式

针对现有系统升级，分析问题和改进点，输出优化版 PRD

🎯
对标开发模式

针对竞品做差异化，分析差异化机会，输出对标版 PRD

✍️
产品经理风格管理

分析你的 PRD 写作风格，创建风格配置，生成符合个人习惯的文档

🎨
设计规范管理

上传和管理设计规范，生成原型时自动应用，保持视觉一致性

🌐
浏览器自动化

内置 Playwright CLI，支持网页截图、自动化测试、数据抓取

Getting Started

安装与启动

3 分钟完成环境准备，开始你的第一个 AI 产品经理项目

准备工作

AI_PM 是一个 Claude Code 技能套件，需要 Claude Code 已安装，且 AI_PM 文件夹在项目根目录。

目录结构要求

确保你的项目目录是这样的：

你的项目/
├── AI_PM/ ← AI_PM 文件夹放在这里
│ ├── .claude/
│ │ └── skills/
│ │ └── ai-pm/
│ │ └── SKILL.md
│ └── ai-pm/
│ └── output/
└── ... 其他文件

启动验证

cd /path/to/your/project
claude

# 验证 AI_PM 是否可用
/ai-pm list
✓ 验证成功

如果看到项目列表（可能是空的），说明安装成功。

🎓 专家级
Style Management

产品经理风格管理

分析你的 PRD 写作风格，生成符合个人习惯的文档

什么是风格管理？

每个人的 PRD 写作风格都不同：有的喜欢用表格，有的喜欢详细描述，有的用 P0/P1/P2 标记优先级，有的用高/中-低。

风格管理功能让 AI 学习你的写作风格，生成符合你习惯的 PRD。

创建你的风格

# 分析你的 PRD 文件，创建风格配置
/ai-pm style analyze ~/Documents/my-prd.md

# AI 会询问风格名称
💬 请为风格命名（如：my-company-style）：

你: my-style

# AI 分析完成后会生成：
# - templates/pm-styles/my-style/style-config.json
# - templates/pm-styles/my-style/examples/original-prd.md

风格分析维度

维度 说明 示例
文档结构 章节组织方式、层级深度 8章结构 / 精简5章
写作风格 语言特征、详细程度 技术严谨 / 产品思维
格式偏好 表格字段、标注方式 P0-P2 / 高中低
内容侧重 用户故事、数据指标比重 详细 / 简要

Design System

设计规范管理

上传和管理设计规范，保持原型视觉一致性

什么是设计规范？

设计规范定义了产品的视觉风格：颜色、字体、组件样式、间距等。

设计规范管理让你可以复用设计规范，确保生成的原型符合你的设计体系。

两种管理方式

类型 存储位置 适用场景
全局设计规范 templates/design-systems/ 跨项目共享（推荐）
项目级设计规范 projects/项目名/08-design-system/ 项目专属风格

创建全局设计规范

# 步骤 1：创建规范文件夹
mkdir -p AI_PM/templates/design-systems/my-company

# 步骤 2：创建规范说明
# 编辑 AI_PM/templates/design-systems/my-company/README.md

风格 vs 设计规范

对比项 产品经理风格 设计规范
关注点 PRD 写作风格、结构、术语 UI 视觉风格、组件、交互
使用时机 生成 PRD 文档时 生成原型时
影响内容 文字描述、章节结构、表格格式 原型视觉、颜色、字体、间距
💡 最佳实践：建议同时配置风格和设计规范，这样生成的 PRD 和原型都能保持一致性。
