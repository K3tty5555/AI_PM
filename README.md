# AI_PM

Claude Code 技能套件，模拟产品经理完整工作流。输入需求，输出 PRD、原型、评审报告。

## 能做什么

- **完整产品流程** — 从一句需求描述出发，依次完成需求分析、竞品研究、用户故事、PRD、可交互原型、六角色评审，每个阶段单独保存文件
- **PRD 多格式导出** — 生成 Markdown / 纯文字 PDF / 含截图 DOCX（可直接导入飞书） / 含截图 PDF，一次生成全套
- **需求优先级评估** — 批量接收各方提报的需求，按业务价值/实现成本/用户影响/战略契合四维打分，输出排序结果和可直发给提报方的回复模板
- **工作周报生成** — 随意描述本周工作，自动归类整理，生成向上汇报版（简洁）或团队同步版（详细）
- **现场调研/客户访谈** — 线下面谈场景，结构化访谈、实时记录，现场生成 PRD
- **数据洞察分析** — 上传 Excel/CSV/JSON 数据文件，挖掘业务洞察，生成交互式可视化仪表盘，提炼产品需求
- **产品分身** — 分析你的历史 PRD，学习写作风格和措辞习惯，让 AI 生成的文档越来越像你写的
- **设计规范管理** — 加载公司 UI 规范，让原型自动遵守你们的设计标准，而不是默认风格
- **产品知识库** — 沉淀设计模式、决策记录、踩坑经验，下次遇到类似问题时自动推荐

## 快速开始

前置条件：[Claude Code](https://claude.ai/code)

```bash
git clone https://github.com/your-repo/AI_PM.git
cd AI_PM
claude  # 打开 Claude Code
```

在 Claude Code 中运行第一条命令：

```bash
/ai-pm "我想做一个记账小程序，帮助年轻人管理日常开支"
```

AI 会引导你完成需求澄清，然后逐步推进到 PRD 和原型。

| `/ai-pm-weekly` | 工作周报生成 | 每周五汇报、项目同步 |
| `/ai-pm-interview` | 现场调研/客户访谈 | 客户拜访、线下调研 |
| `/ai-pm-data` | 数据洞察分析与仪表盘 | 分析用户行为数据、发现产品机会 |
| `/ai-pm-persona` | 产品分身（写作风格学习） | 让 AI 模仿你的 PRD 风格 |
| `/ai-pm-design-spec` | 设计规范管理 | 加载公司 UI 规范、统一原型风格 |
| `/ai-pm-knowledge` | 产品知识库 | 沉淀经验、搜索历史决策 |
| `/ai-pm-analyze` | 需求分析 | 单独运行分析阶段 |
| `/ai-pm-research` | 竞品研究 | 市场对标、差异化机会 |
| `/ai-pm-story` | 用户故事编写 | 生成用户故事和验收标准 |
| `/ai-pm-prd` | PRD 生成与导出 | 生成 MD / PDF / DOCX / 飞书格式 |
| `/ai-pm-prototype` | 原型生成 | 生成可交互 HTML 原型 |
| `/ai-pm-review` | 需求评审（九角色） | PRD 评审、多轮迭代 |
| `/agent-team` | 多代理并行引擎 | 大项目并行处理，通常由 `/ai-pm --team` 触发 |
| `/tutorial-center-update` | 更新教程中心 HTML | 新增技能后同步教程 |

## PRD 模板

项目内置 PRD 写作模板，位于 `templates/prd-styles/default/feishu-template.md`，包含标准八章结构和字段规范。可直接替换为团队自定义模板，生成时自动读取并严格对齐。
