# 用户故事编写

## 输入

- 主要：`{项目目录}/02-analysis-report.md`（需求分析）
- 参考：`{项目目录}/03-competitor-report.md`（竞品研究，如有）

## 输出

`{项目目录}/04-user-stories.md`（用户故事文档）

## 执行步骤

### 步骤1：读取分析文档，规划模块

从需求分析中提取用户角色、核心功能、优先级，划分 Epic 模块。

### 步骤2：询问用户补充特殊场景

```
基于需求分析，计划编写以下模块的用户故事：
{模块列表}

在开始前，有什么特殊场景需要覆盖？例如：
- 网络异常时的处理（断网、超时）
- 用户误操作容错（误删、误提交）
- 权限相关限制（无权限、权限变更）
- 数据边界情况（空值、超长、特殊字符）

请告诉我，或回复"按标准编写"。
```

### 步骤3：编写用户故事

**标准格式：**
```
作为 [用户角色]，
我想要 [功能/能力]，
以便 [价值/目标]。
```

**每个故事必须包含：**
- 故事编号：US-{模块缩写}-{序号}（如 US-AUTH-001）
- 优先级：P0/P1/P2
- 验收标准：Given/When/Then 格式，覆盖正常流程、异常流程、边界情况

**按功能模块分组，同模块内按优先级排序（P0 先）。**

### 步骤4：输出汇总表

完成后输出故事优先级矩阵，便于迭代规划。
