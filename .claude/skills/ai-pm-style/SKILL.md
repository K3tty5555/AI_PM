# 产品经理风格分析

## 执行协议

- 深度分析用户上传的 PRD 文档
- 提取可复用的风格特征
- 生成结构化的风格配置文件
- 支持风格对比和选择建议

## 风格分析维度

### 1. 文档结构偏好

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| 章节顺序 | PRD 的章节组织方式 | 标准8章/精简5章/自定义 |
| 表格密度 | 使用表格的频率和复杂度 | 高频/适中/低频 |
| 层级深度 | 标题层级使用情况 | 3层/4层/5层+ |
| 必填章节 | 哪些章节是必须的 | 文档概述+详细设计 |

### 2. 写作风格特征

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| 语言风格 | 描述方式 | 技术严谨/产品思维/业务导向 |
| 详细程度 | 内容详尽度 | 极简/标准/详尽 |
| 术语习惯 | 专业术语使用 | 互联网术语/传统软件术语/自定义 |
| 视角 | 叙述角度 | 用户视角/系统视角/混合 |

### 3. 格式偏好

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| 字段规范 | 表格字段设置 | 标准字段/扩展字段/自定义 |
| 标注方式 | 重点内容标注 | 加粗/引用/高亮块 |
| 流程表达 | 流程图形式 | 文字描述/Mermaid/伪代码 |
| 优先级标记 | 优先级表示 | P0-P2/高中低/必须-应该-可以 |

### 4. 内容侧重

| 维度 | 分析内容 | 示例值 |
|------|---------|--------|
| 用户故事 | 用户场景描述比重 | 重/中/轻 |
| 数据指标 | 量化指标关注 | 详细/简要/忽略 |
| 异常处理 | 边界情况覆盖 | 全面/主要/简单 |
| 技术细节 | 实现细节描述 | 详细/适度/不涉及 |

## 执行流程

### 场景1：分析新的 PRD 并创建风格

```
用户上传 PRD 文件
    ↓
读取 PRD 文档
    ↓
分析文档结构
    • 识别章节组织
    • 统计表格使用情况
    • 分析层级深度
    ↓
分析写作风格
    • 语言特征提取
    • 术语习惯识别
    • 详细程度评估
    ↓
分析格式偏好
    • 表格字段提取
    • 标注方式识别
    • 优先级标记方式
    ↓
提取示例片段
    • 每个章节选取代表性文本
    ↓
生成 style-config.json
    ↓
保存到 templates/pm-styles/{风格名}/
```

### 场景2：列出已有风格

```
扫描 templates/pm-styles/
    ↓
读取每个风格的 style-config.json
    ↓
生成风格列表
    ↓
展示风格特征摘要
```

### 场景3：预览风格效果

```
读取风格配置
    ↓
生成示例 PRD 片段
    ↓
展示应用该风格后的效果
```

## 命令支持

| 命令 | 作用 | 示例 |
|------|------|------|
| `analyze {文件路径}` | 分析 PRD 生成风格 | `analyze /path/to/prd.md` |
| `list` | 列出所有风格 | `list` |
| `preview {风格名}` | 预览风格效果 | `preview my-style` |
| `delete {风格名}` | 删除风格 | `delete old-style` |
| `export {风格名}` | 导出风格配置 | `export my-style` |

## 与主流程整合

风格配置在 PRD 生成时被读取：

```
ai-pm-prd 技能执行时
    ↓
检查是否指定了风格
    ↓
读取 templates/pm-styles/{风格名}/style-config.json
    ↓
根据配置调整：
    • 章节结构
    • 表格格式
    • 术语使用
    • 详细程度
    ↓
生成符合风格的 PRD
```
