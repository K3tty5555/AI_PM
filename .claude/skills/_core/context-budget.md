# Token 预算管理策略

## 当前限制
- **模型上限**: 262,144 tokens
- **安全阈值**: 200,000 tokens (保留 62k 缓冲)
- **紧急截断线**: 240,000 tokens

## 预算分配 (总计 200k)

| 类别 | 预算 | 内容 |
|-----|------|------|
| 系统提示 | 10k | 基础指令、工具定义 |
| 技能核心 | 30k | SKILL.md + 当前阶段需要的子文件 |
| 记忆文件 | 20k | MEMORY.md (前200行) + 活跃主题 |
| 项目上下文 | 80k | 项目文件、PRD、原型 |
| 对话历史 | 50k | 最近 10 轮对话 (超则压缩) |
| 缓冲区 | 10k | 临时计算、响应空间 |

## 当接近阈值时的策略

### Level 1: 180k (预警)
- 停止自动加载非核心技能文件
- 使用文件引用代替内容内联

### Level 2: 200k (限制)
- 只保留最近 5 轮对话
- 大文件转为摘要模式

### Level 3: 240k (紧急)
- 主动提议开启新会话
- 保存当前状态到 checkpoint 文件
