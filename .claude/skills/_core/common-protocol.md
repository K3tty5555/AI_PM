# AI_PM 技能公共执行协议

所有 AI_PM 系列技能遵循以下执行协议。

## 一、前置检查

执行任何技能前，自动检查：

1. **项目上下文**
   - 当前是否在项目目录下？
   - 是否有 `.current-project` 标记？

2. **依赖产出物**
   - 前置阶段是否已完成？
   - 依赖文件是否存在？

3. **用户确认**
   - 是否明确用户意图？
   - 是否需要补充信息？

## 二、执行流程

```yaml
Phase 1: 初始化
  - 读取项目配置
  - 检查前置条件
  - 确认执行参数

Phase 2: 执行
  - 按协议执行任务
  - 实时反馈进度
  - 处理异常情况

Phase 3: 交付
  - 生成产出物
  - 质量自检
  - 用户确认
```

## 三、产出物规范

### 文件名规范

```
{序号}-{英文名称}.md

示例：
01-requirement-draft.md
05-PRD-v1.0.md
08-review-report-v1.md
```

### Markdown 规范

- 使用标准 Markdown 语法
- 标题层级：#/##/###
- 表格对齐，表头使用 |---|---|
- 代码块标注语言

## 四、质量门禁

每个阶段必须通过质量检查：

- [ ] 文档结构完整
- [ ] 格式规范统一
- [ ] 术语使用一致
- [ ] 交叉引用正确

## 五、错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 前置条件不满足 | 提示用户完成前置阶段 |
| 文件读取失败 | 重试3次，失败后人工介入 |
| 用户取消 | 保存当前进度，优雅退出 |
| 未知错误 | 记录日志，提供反馈渠道 |

## 六、与其他技能协作

### 数据流向

```
analyze → research → story → prd → prototype → review
    ↓                                    ↓
knowledge (自动沉淀)              knowledge (自动沉淀)
```

### 配置依赖

- 读取 `templates/prd-styles/` 写作风格
- 读取 `templates/ui-specs/` UI规范
- 读取项目 `.ai-pm-config.json` 配置
