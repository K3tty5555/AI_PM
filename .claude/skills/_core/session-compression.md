# 会话压缩协议

## 何时压缩

- 对话轮数 > 10
- Token 使用 > 180k
- 用户主动要求简化

## 压缩策略

### 1. 对话摘要格式

```markdown
## 会话摘要 [R1-R5]

### 关键决策
- D1: 目标用户 = B端企业用户
- D2: 核心痛点 = 跨部门协作效率低
- D3: MVP范围 = 消息+任务+日历

### 已完成
- ✅ Phase 1: 需求澄清
- ✅ Phase 2: 需求分析
- ✅ Phase 3: 竞品研究

### 进行中
- 🔄 Phase 4: 用户故事编写

### 待确认
- Q1: 是否需要集成钉钉?
- Q2: 数据导出格式偏好?

### 上下文Token
- 已压缩 5 轮对话
- 当前Token: ~185k
```

### 2. 文件引用规则

```markdown
❌ 不要: 粘贴PRD全文
✅ 要: "详见 projects/xxx/05-prd/README.md"

❌ 不要: 列出所有用户故事
✅ 要: "4个核心用户故事已完成，详见 04-user-stories.md"

❌ 不要: 展示原型代码
✅ 要: "原型已生成: projects/xxx/06-prototype/index.html"
```

### 3. 自动截断清单

当 Token > 200k，自动执行：

1. **立即执行**
   - 保存 checkpoint
   - 提示用户新开会话

2. **当前会话进入压缩模式**
   - 只回复核心问题
   - 不主动加载技能文件
   - 最小化回复长度

3. **新会话恢复**
   ```
   "继续项目 xxx 的工作，已加载 checkpoint。"
   自动读取: .checkpoint.md
   ```

## 用户提示模板

### Token 预警 (180k)
```
⚠️ 上下文即将满载 (90%)，建议:
1. 完成当前阶段后新开会话
2. 我会自动保存进度到 checkpoint
```

### Token 紧急 (200k)
```
🔴 上下文满载！正在保存状态...

已保存到: projects/xxx/.checkpoint.md

请开启新会话，输入:
/ai-pm load xxx

我将自动恢复当前进度。
```
