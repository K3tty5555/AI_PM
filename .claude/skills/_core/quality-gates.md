# AI_PM 质量门禁标准

## 需求阶段质量门禁

### 检查项

- [ ] **目标用户画像清晰**
  - 有明确的用户群体定义
  - 包含年龄/职业/场景等关键信息

- [ ] **核心场景描述完整**
  - 至少3个核心使用场景
  - 每个场景包含：用户、行为、目标

- [ ] **痛点分析到位**
  - 明确当前痛点
  - 量化痛点影响

- [ ] **功能范围明确**
  - MVP功能清晰
  - 明确排除的功能

- [ ] **优先级合理**
  - P0/P1/P2分级明确
  - P0功能不超过5个

## 架构阶段质量门禁

- [ ] **架构图清晰**
  - 模块边界明确
  - 依赖关系清晰

- [ ] **技术选型有理由**
  - 每个选型有对比分析
  - 考虑团队能力

- [ ] **扩展性考虑**
  - 支持未来功能扩展
  - 性能预留空间

- [ ] **风险识别**
  - 列出关键风险
  - 有应对措施

## 产出物质量门禁

### PRD 文档

- [ ] 文档结构完整（8章标准结构）
- [ ] 逻辑清晰无矛盾
- [ ] 功能描述详细
- [ ] 流程图准确
- [ ] 边界情况考虑周全
- [ ] 术语定义一致

### 原型

- [ ] 符合PRD描述
- [ ] 交互逻辑完整
- [ ] 响应式适配
- [ ] 视觉风格统一
- [ ] 交互动效流畅

### 数据分析

- [ ] 数据完整性检查
- [ ] 指标口径定义清晰
- [ ] 埋点覆盖完整
- [ ] 可视化准确无误

## 评审标准

| 等级 | 定义 | 处理方式 |
|-----|-----|---------|
| Critical | 必须修改 | 阻塞后续流程 |
| Major | 建议修改 | 需确认是否修改 |
| Minor | 可选修改 | 记录，可后续优化 |

## 自动化检查

### 可自动检查
- 文档结构完整性
- 必填字段存在性
- 格式规范性

### 需人工检查
- 业务逻辑合理性
- 用户体验流畅度
- 创新性评估
