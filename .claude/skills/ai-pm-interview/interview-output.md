### 合并到项目主目录

**触发条件**：用户输入 `客户满意` 并选择 `导出PRD` / `导出原型` / `全部导出` 时执行。

**合并操作**：
1. 将 PRD 草案最终版复制到 `05-prd/` 目录，按版本号递增命名（如 `05-PRD-v1.1.md`）
2. 将原型最终版复制到 `06-prototype/` 目录
3. 更新项目主原型的侧栏导航（如有新页面）
4. 在访谈 `00-meta.json` 中标记 `"merged": true`

**合并交互**：
```
📦 准备合并到项目主目录

将要执行：
   • 04-prd-draft-v2.md → 05-prd/05-PRD-v1.1.md
   • 05-prototype-v2/   → 06-prototype/pages/

⚠️ 主目录现有文件不会被覆盖，新版本将使用递增版本号。

💬 确认合并？（是/否）
```

**注意**：草案目录中的原始文件保留不删除，作为历史记录。

## 快捷指令汇总

### 全局指令（任意阶段可用）

| 指令 | 作用 |
|-----|------|
| `帮助` / `help` | 显示可用指令 |
| `状态` | 显示当前访谈状态 |
| `保存` | 手动保存进度（自动保存到当前访谈目录） |
| `退出` / `exit` | 退出并自动保存（支持断点续传） |
| `列表` / `list` | 查看所有访谈列表 |
| `切换 {ID}` | 切换到其他访谈 |

### 入口指令（启动时使用）

| 指令 | 作用 |
|-----|------|
| `继续` / `resume` | 恢复未完成的访谈（有进行中的访谈时） |
| `新建` / `new` | 开始新访谈 |
| `删除 {ID}` | 删除指定访谈 |

### 访谈阶段指令

| 指令 | 作用 |
|-----|------|
| `下一题` / `next` | 下一问题 |
| `上一题` / `prev` | 上一问题 |
| `跳过` / `skip` | 跳过当前 |
| `追问` | 追问建议 |
| `总结` | 访谈总结 |
| `完成` | 结束访谈 |

### 迭代阶段指令

| 指令 | 作用 |
|-----|------|
| `客户满意` | 确认方案 |
| `修改 xxx` | 修改方案 |
| `返回访谈` | 回退到访谈 |
| `查看草图` | 查看原型 |
| `对比版本` | 版本对比 |

### 访谈管理指令（断点续传）

| 指令 | 作用 |
|-----|------|
| `继续` / `resume` | 恢复未完成的访谈 |
| `新建` / `new` | 开始新访谈 |
| `列表` / `list` | 查看所有访谈 |
| `切换 {访谈ID}` | 切换到指定访谈 |
| `删除 {访谈ID}` | 删除访谈 |

## 与 Agent Team 的协作

现场调研模式可与 **Agent Team** 深度协作，利用专属角色提升调研效率：

### 专用角色

| 角色 | 职责 | 协作阶段 |
|-----|------|---------|
| 🎤 **用户研究员** | 执行结构化访谈、实时记录、洞察分析 | Phase 1 |
| 🧩 **需求整合师** | 提取需求、整合信息、优先级排序 | Phase 2 |

### 使用方式

```bash
# 启动现场调研专用团队
/agent-team --mode=serial \
  --roles=user-researcher,requirement-synthesizer,pm,designer \
  "后台管理系统-订单模块现场调研"

# 多客户并行调研
/agent-team --mode=parallel \
  --roles=user-researcher,user-researcher,requirement-synthesizer,pm \
  "CRM系统用户体验调研"
```

### 协作流程

```
用户研究员
    ↓ 执行访谈
    输出：访谈记录、用户画像、洞察报告

需求整合师
    ↓ 分析访谈记录
    输出：需求清单、PRD草案

产品经理
    ↓ 完善PRD
    输出：完整PRD

UI设计师
    ↓ 设计原型
    输出：可交互原型
```
