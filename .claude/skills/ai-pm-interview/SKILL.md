# ai-pm-interview — 现场调研技能

## 适用场景

- 已有产品需要改进，需现场了解用户痛点
- 线下拜访客户，面对面收集反馈
- 需要快速产出需求草稿当场确认

**与标准模式的区别**: 现场模式优先速度，输出精简版草稿；标准模式输出完整文档。

---

## 访谈流程

### Phase 0：准备（访谈前）

1. 询问调研主题（功能名或业务场景）
2. 了解被访者角色（内部业务方 / 外部客户）
3. 输出 2-3 个核心提问方向

---

### Phase 1：现场访谈

按以下结构逐步提问，**每次只问一个问题**：

**开场**（建立信任，了解背景）
```
1. 您目前怎么完成这件事？
2. 一周/一天用多少次？
```

**现状了解**（摸清使用习惯）
```
3. 哪个步骤最花时间？
4. 有没有用其他工具辅助？
```

**痛点挖掘**（找到真实需求）
```
5. 遇到过什么让您觉得"这个真烦"的情况？
6. 如果这个问题解决了，对您影响有多大？
```

**需求确认**（验证解决方向）
```
7. 您理想中这个功能应该怎么运作？
8. 有没有见过类似产品做得好的？
```

**优先级排序**（聚焦最重要的）
```
9. 如果只能改一件事，您最希望改哪个？
```

---

### Phase 2：实时记录

访谈过程中同步记录，格式：

```markdown
## 访谈记录 — {功能名} — {日期}

**被访者**: {角色/部门}
**时长**: {分钟}

### 用户原话（直接引用）
- "{原话1}"
- "{原话2}"

### 痛点
- {痛点描述}（频率：高/中/低）

### 期望
- {期望描述}

### 优先级
1. {最重要的需求}
2. {次重要的需求}
```

记录原则：
- 记用户说的话，不记自己的解读
- 标注频率（每天/每周/偶尔）
- 记下具体数字（"等了3分钟"比"等很久"有用）

---

### Phase 3：快速生成需求草稿

访谈结束后立即执行：

1. 整理访谈记录，提炼核心需求（3-5条）
2. 生成需求草稿，格式：

```markdown
## 需求草稿 — {功能名}

### 背景
{一句话说明用户是谁、在做什么、遇到什么问题}

### 核心需求
| 需求 | 用户原话依据 | 优先级 |
|------|------------|--------|
| {需求1} | "{原话}" | 高 |
| {需求2} | "{原话}" | 中 |

### 暂缓/不做
- {不在本次范围的需求}
```

3. 展示给客户确认，收集反馈
4. 根据反馈调整，最多迭代 2 轮

---

### Phase 4：生成正式 PRD

客户确认需求草稿后：

```
已确认需求，现在生成正式 PRD
→ 触发 ai-pm-prd 流程，输入为当前访谈记录 + 需求草稿
```

输出到 `output/projects/{项目名}/` 目录。

---

## Anti-Pattern

- 不在现场写详细文档，草稿够用即可
- 不一次性问完所有问题再记录，边问边记
- 不过度解读客户需求（记原话，别加工）
- 不承诺无法实现的交付时间
- 生成内容要快，客户不能等超过30秒
