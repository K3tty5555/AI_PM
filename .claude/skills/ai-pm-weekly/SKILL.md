## 角色说明

帮你把散乱的工作记录整理成专业周报，无需提前整理格式，随意描述即可。
如果配置了产品分身（ai-pm-persona），自动应用你的写作风格，让周报更像你写的。

## 执行流程

1. 引导用户描述本周工作，明确告知：可以随意描述，不需要任何格式
2. 如信息不足，追问一次补充：本周有什么亮点/阻碍/风险？下周重点是什么？
3. 自动将内容归类：已完成 / 进行中 / 下周计划 / 风险依赖
4. 检查 `templates/prd-styles/` 是否配置了产品分身风格，有则应用措辞习惯
5. 按选定模板（`--detail` 或 `--brief`）输出周报
6. 保存到输出文件

## 产品分身集成

- 检查 `templates/prd-styles/` 是否有当前生效的风格配置
- 有则参照该风格的措辞习惯（用词、句式、简洁度）生成周报
- 无则使用标准中性专业风格

## 输出格式：团队同步版（`--detail`，默认）

```
# 周报 - {姓名} - {起始日期} ~ {结束日期}

## 本周完成
- {已完成事项，带项目/背景说明}

## 进行中
- {进行中事项} — {当前进度 / 下一步行动}

## 下周计划
- {计划事项}

## 风险与依赖
- {风险项}（如无则省略此节）
```

## 输出格式：向上汇报版（`--brief`）

```
本周重点：{1-2句话核心进展}

完成：
- {条目，每条一行，简洁}
- （3-5条）

进行中：
- {1-2条最重要的}

下周：
- {2-3条最重要的}
```

## 输出文件路径

所有周报统一保存到：`output/weekly/周报-{YYYY-MM-DD}.md`

日期取周报所属周的周五日期（或当天日期，如当天即为周五）。
目录不存在时自动创建（`Bash(mkdir)`）。
