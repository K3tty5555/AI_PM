# Agent Team 体验调优流程

## 调优目标

对生成的 HTML 教程中心进行多维度体验优化，确保：
- 视觉设计专业一致
- 文案内容准确清晰
- 用户体验流畅自然

## 调优流程

### Round 1: UI Designer 视觉评审

**角色**: UI Designer (ui-designer)
**职责**: 检查视觉设计是否符合规范

**检查清单**:
- [ ] 颜色系统 - 是否使用规范中的 CSS 变量
- [ ] 字体系统 - 大小、字重、行高是否正确
- [ ] 间距系统 - 是否遵循 8pt 网格
- [ ] 圆角系统 - 卡片、按钮圆角是否一致
- [ ] 阴影系统 - 层次是否清晰
- [ ] 动画效果 - 缓动函数、持续时间是否规范
- [ ] 响应式 - 各断点布局是否正确
- [ ] 暗色模式 - 切换是否正常，颜色是否正确

**输出**: 视觉优化建议报告

### Round 2: Tech Writer 文案评审

**角色**: Tech Writer (tech-writer)
**职责**: 检查文案质量和准确性

**检查清单**:
- [ ] 技能名称 - 是否简洁有力
- [ ] 简短描述 - 是否准确传达功能
- [ ] 使用场景 - 是否覆盖高频/痛点/入门场景
- [ ] 示例案例 - 是否真实可信、有代入感
- [ ] 命令格式 - 是否准确、可执行
- [ ] 一致性 - 相同概念是否表述一致
- [ ] 语气语调 - 是否友好专业
- [ ] 错别字 - 是否有错别字或语病

**输出**: 文案优化建议报告

### Round 3: Product Manager 体验评审

**角色**: Product Manager (product-manager)
**职责**: 检查整体用户体验

**检查清单**:
- [ ] 信息架构 - 技能分类是否合理
- [ ] 导航逻辑 - 首页→详情→返回是否顺畅
- [ ] 首屏体验 - 首屏是否传达核心价值
- [ ] 技能发现 - 用户能否快速找到所需技能
- [ ] 学习曲线 - 新用户能否理解如何使用
- [ ] 交互反馈 - 点击、悬停是否有明确反馈
- [ ] 性能体验 - 加载、切换是否流畅
- [ ] 可用性 - 所有功能是否正常工作

**输出**: 体验优化建议报告

## 评审方式

### 并行评审

```yaml
mode: parallel
agents:
  - ui-designer
  - tech-writer
  - product-manager
```

同时启动三个代理，各自独立评审，最后汇总建议。

### 串行评审

```yaml
mode: serial
order:
  1. ui-designer    # 先确保视觉正确
  2. tech-writer    # 再优化文案
  3. product-manager # 最后整体体验
```

推荐串行模式，每轮根据上一轮建议修改后再进入下一轮。

## 评审输入

### 提供给 Agent 的信息

1. **生成的 HTML 文件路径**
   - `AI_PM_教程中心.html`

2. **设计规范**
   - `.claude/skills/tutorial-center-update/generator.md` - HTML 生成器规范（CSS 变量、动画系统）
   - `.claude/skills/tutorial-center-update/SKILL.md` - 内容规范（文案风格、技能卡片规范）

3. **所有 Skill 源文件**
   - `.claude/skills/*/SKILL.md`

4. **评审模板**

   ```markdown
   ## {角色} 评审报告

   ### 通过项
   - [x] 项目1
   - [x] 项目2

   ### 建议项
   - [ ] 问题描述
     - **位置**: 具体位置
     - **问题**: 详细说明
     - **建议**: 如何改进
     - **优先级**: P0/P1/P2

   ### 总评
   - 评分: X/10
   - 是否通过: ✅/❌
   - 关键问题: 如有
   ```

## 优化循环

```
生成 HTML
    ↓
UI Designer 评审
    ↓ (修改)
Tech Writer 评审
    ↓ (修改)
Product Manager 评审
    ↓ (修改)
终版输出
```

每轮评审后：
1. 收集所有建议
2. 按优先级排序 (P0 > P1 > P2)
3. 修改 HTML
4. 进入下一轮

## 终止条件

满足以下任一条件即可输出终版：

1. **全票通过** - 三个角色都给出 ≥8 分
2. **无 P0 问题** - 仅剩 P1/P2 优化建议
3. **达到最大轮数** - 最多 3 轮优化

## 最终输出

优化完成后的交付物：

1. **AI_PM_教程中心.html** - 最终版本
2. **review-report.md** - 评审报告汇总
   - 各角色评分
   - 采纳的建议
   - 未采纳的建议及原因

## 使用示例

```bash
# 完整流程：生成 + 调优
/tutorial-center-update

# 仅调优（已有 HTML）
/tutorial-center-update review

# 快速模式（跳过 Agent 评审）
/tutorial-center-update quick
```
