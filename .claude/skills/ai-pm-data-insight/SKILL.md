# 数据洞察与需求发现

> ⚠️ **已合并**: 本技能已合并到 `ai-pm-data`，请使用 `/ai-pm data insight`

## 定位

AI_PM 的**数据驱动入口**，让数据成为产品需求的源头。

## 迁移提示

```
旧命令: /ai-pm data-insight
新命令: /ai-pm data insight

过渡期: 2026-03-01 ~ 2026-03-15 (2周)
```

- **数据探索**: 上传数据文件，自动进行探索性分析
- **洞察发现**: 从数据中识别用户行为模式、业务机会
- **需求提炼**: 将数据洞察转化为产品需求
- **流程驱动**: 可直接触发ai-pm完成产品全流程

## 与其他技能的关系

```
用户上传数据文件
    ↓
ai-pm-data-insight 【本技能】
    • 数据解析与清洗
    • 探索性分析(EDA)
    • 洞察发现与交互
    • 需求提炼
    ↓
可选择继续执行:
    ├─→ ai-pm-analytics  ← 设计指标体系（基于真实数据分布）
    ├─→ ai-pm-analyze    ← 深入需求分析
    ├─→ ai-pm-prd        ← 直接生成PRD
    └─→ ai-pm (主控)     ← 完整产品流程
```

## 支持的数据格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Excel | .xlsx, .xls | 多sheet支持，自动识别表头 |
| CSV | .csv | 标准逗号分隔，支持大文件 |
| JSON | .json | 结构化数据，支持嵌套解析 |
| Parquet | .parquet | 列式存储，高性能分析 |
| SQL导出 | .sql | 数据库导出文件 |

### Phase 3: 洞察发现与交互

**关键洞察汇总:**

```
📈 数据洞察报告

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 关键发现 (直接影响业务)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 【严重】首页→列表页转化率仅28%
   影响: 72%的用户在首页流失
   根因推测: 首页信息过载、导航不清晰、CTA不明显
   产品机会: 首页重构、个性化推荐

2. 【严重】搜索结果质量差
   影响: 用户搜索后快速离开
   根因推测: 搜索算法不准确、结果排序不合理
   产品机会: 搜索优化、智能推荐

3. 【重要】周末活跃度低35%
   影响: 用户粘性不足
   根因推测: 缺乏周末使用场景
   产品机会: 周末活动、内容运营

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟡 优化机会 (提升体验)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4. 收藏功能使用率仅23%
   → 可能入口太深，建议优化交互

5. 新用户7日留存低于老用户
   → 新手引导可能需要优化

6. 移动端占比68%，但页面深度低于PC
   → 移动端体验可能需要优化
```

当前首页采用固定布局，所有用户看到相同内容。

• 新用户: 67%点击"新手指引"

设计个性化首页，根据用户画像动态调整内容：
• 新用户: 突出新手指引和热门内容
• 活跃用户: 突出搜索和历史记录
• 高价值用户: 突出专属活动和优惠

【预期效果】
• 首页转化率从28%提升至40%
• 用户停留时长增加20%
• 整体GMV提升15%

【优先级】P0

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 搜索使用率: 78% (高)
• 搜索后平均停留: 12秒 (偏低)
• 搜索→详情页转化率: 18%

【问题诊断】
用户频繁搜索但快速离开，说明搜索结果不符合预期。

【产品需求】
优化搜索算法，增加个性化排序因子：
• 基于用户历史行为的个性化排序
• 增加"智能推荐"搜索联想
• 结果页增加筛选和排序选项

【预期效果】
• 搜索停留时长提升至25秒
• 搜索转化率提升至30%

【优先级】P0

**技术栈：**
- **图表库**: ECharts 5.x（专业数据可视化）
- **样式**: Tailwind CSS + 自定义主题
- **交互**: Vanilla JavaScript（无框架依赖）
- **响应式**: 支持桌面端、平板、手机
- **主题**: 支持暗/亮主题切换

**核心功能：**
1. **实时筛选** - 时间范围、用户群体、功能模块多维筛选
2. **图表联动** - 点击图表元素，其他图表自动联动过滤
3. **下钻分析** - 从汇总数据下钻到明细数据
4. **洞察卡片** - 结构化展示关键发现，支持展开详情
5. **原始数据** - 提供数据表格预览和导出
6. **一键导出** - 支持 PDF 报告导出
7. **分享功能** - 生成分享链接（导出静态HTML）

## 命令体系

| 命令 | 作用 | 示例 |
|------|------|------|
| `/ai-pm data-insight {文件路径}` | 分析数据文件 | `data-insight ./data.xlsx` |
| `/ai-pm data-insight --focus=conversion` | 聚焦转化率分析 | `data-insight ./data.csv --focus=conversion` |
| `/ai-pm data-insight --format=csv` | 指定文件格式 | `data-insight ./data --format=csv` |
| `/ai-pm data-insight --visualize` | 分析并生成交互式可视化报告 | `data-insight ./data.xlsx --visualize` |
| `/ai-pm data-insight dashboard` | 仅生成可视化仪表盘（基于已有分析） | `data-insight dashboard` |

## 与ai-pm-analytics的协作

```
ai-pm-data-insight (发现洞察)
    ↓ 输出数据特征和业务洞察
ai-pm-analytics (设计指标)
    ↓ 基于真实数据设计指标体系
ai-pm-prd (生成文档)
    ↓ 包含数据洞察支撑的需求
ai-pm-prototype (原型)
    ↓ 可集成数据看板预览
```

## Anti-Pattern

- ❌ 不要过度解读数据相关性为因果性
- ❌ 不要忽视样本偏差（如只分析活跃用户）
- ❌ 不要用平均值掩盖分布特征
- ❌ 不要基于短期波动做长期决策
- ✅ 要结合定性研究验证数据洞察
- ✅ 要明确数据的时间范围和代表性

## 核心理念

> **让数据说话，但不让数据替你做决定。**

数据洞察是产品决策的重要输入，但不是唯一输入。优秀的PM需要结合用户访谈、竞品分析、技术可行性等多维度信息，做出综合判断。
