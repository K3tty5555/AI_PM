# 常见场景速查 + Anti-Pattern

## 常见场景速查

| 用户说的话 | 处理方式 |
|-----------|---------|
| "PRD 太长了" | 按模块拆分，建议分文档管理 |
| "原型和 PRD 对不上" | 触发 Phase 7.5 审计，比对功能覆盖率 |
| "上次做到哪了" | 读取 _status.json 恢复到 last_phase 的下一步 |
| "不想做竞品研究" | 可跳过，标记 phases.competitor = "skipped"，后续阶段正常推进 |
| "换个风格" | 重新选择 PRD 风格（`/ai-pm config style`）或原型设计规范（`/ai-pm config ui`） |
| "需求改了" | 回到 Phase 1 需求澄清，标记变更点，后续阶段重新执行 |
| "生成的 PRD 质量不好" | 对照样例 PRD（`templates/prd-styles/default/sample-*.md`），检查字段完整性 |
| "导出格式有问题" | 检查 pdf-style.css 排版配方和 PingFang SC 字体可用性 |
| "评审意见太多" | 按严重程度排序（必须改 > 建议改 > 可忽略），逐项让用户决定 |
| "直接出 PRD" | 进入 Yolo 模式，Phase 1-4 自动执行，PRD 前停一次确认 |
| "帮我做个原型" | 检查 PRD 是否存在，存在则进入 Phase 7，不存在则提示先生成 PRD |
| "把截图插入 PRD / DOCX" | 触发 Phase 7.6：先扫描 6 所有表格输出插入计划、等用户确认，再做 manifest 预检，最后调用 md2docx.py 导出 |
| "PRD 里有些表格没有原型图" | 执行 Phase 7.5 6 覆盖检查，逐表确认哪些有界面、哪些无界面，补全 `原型示意` 行或加 `无界面交互` 备注 |
| "我有现成文档" | 引导放入 `07-references/`，读取后跳过 Phase 1 访谈 |

## Anti-Pattern

- 每次只问 1-2 个最重要的问题，不一次抛出所有
- 不假设用户懂产品术语，必要时解释
- 不在用户未确认时跨越关键节点
- 不混用不同项目的文件
- 不生成无法落地的 PRD
- 用户有现成文档时，引导放入 07-references/，不让用户直接发文件到对话框
