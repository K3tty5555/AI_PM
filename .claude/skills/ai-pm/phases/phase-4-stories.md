## 一、用户故事（所有产品类型必写）

### 1.1 写法

```
US-01 作为 {用户角色}，
      我希望 {功能描述}，
      以便 {价值/目的}。

验收标准：
- [ ] {可量化的验收条件}
- [ ] ...
```

### 1.2 INVEST 质量门控（落盘前执行）

写完用户故事后，立即对每条做 INVEST 自检：

| 维度 | 检查问题 | 不达标标准 |
|------|---------|-----------|
| **I**ndependent | 此故事可以独立开发吗？ | 强依赖另一条未完成故事 |
| **N**egotiable | 这是约束还是解决方案？ | 已固化了技术实现方式 |
| **V**aluable | 对用户有明确价值吗？ | 「以便」部分是空话或无法量化 |
| **E**stimable | 能估算工作量吗？ | 描述过于模糊，无法给出工作日估算 |
| **S**mall | 一个 sprint 内能完成吗？ | 验收标准超过 5 条 |
| **T**estable | 验收标准可以被测试验收吗？ | 包含「系统应该」「用户应该」等模糊表述 |

### 1.3 自检结果展示

```
📋 用户故事质量检查（INVEST）

✅ US-01 作为教师，我希望查看考试成绩分布...
   I✅ N✅ V✅ E✅ S✅ T✅

⚠️ US-03 作为管理员，我希望系统有良好的性能...
   I✅ N✅ V⚠️（「良好」无法量化） E⚠️ S✅ T⚠️（无可测试验收标准）

❌ US-07 作为用户，我希望使用微服务架构...
   N❌（固化了技术实现）

通过：{N} 条，待改进：{M} 条，不达标：{K} 条
```

- 全部通过（M+K=0）→ 静默继续
- 有待改进/不达标 → 询问：「继续（带瑕疵进入 PRD）/ 修改」；用户选修改 → 引导改完后重新自检

## 二、Agent 故事（仅 agent / hybrid 必写）

> **核心思路**：把 AI 当一个"虚拟员工"叙事描述——它什么时候出场、怎么帮用户做事、遇到问题怎么办、这个故事反推出哪些产品决策。
>
> **PM 视角**：不是写算法侧的 prompt 设计，是写**产品决策**——AI 应该怎么表现、什么时候降级、什么时候转人工。具体 prompt 文案由算法侧实现，写不出的标 `[算法补完]`。

### 2.1 写法（4 段必填 + 反模式）

```
## AS-01 {场景标题，简洁有画面感}

**触发**：{用户在什么页面 / 什么状态 / 输入什么 → AI 出场}
**AI 怎么做**：
  1. {第一步：理解 / 复述 / 拉数据}
  2. {第二步：处理 / 决策 / 生成}
  3. {第三步：呈现 / 等用户反馈}
  4. {第四步：落地动作 / 跳转 / 写库}
**遇到问题怎么办**：
  - {失败场景 1}：{AI 给的降级方案}
  - {失败场景 2}：{AI 给的兜底动作}
**说明的产品决策**：
  - {从这个故事能反推出至少 1 条 PM 决策}
  - {例：必须先复述再生成 / 改细节用对话不用表单 / 失败要主动给方案不报错}
```

### 2.2 例子（教育场景）

```
## AS-01 老师一句话组卷场景

**触发**：老师在备课页打开 AI 助手，输入"给二班出一份高三模拟数学卷，对标 2024 北京高考"
**AI 怎么做**：
  1. 听懂意图 → 复述 5 行摘要（卷型/对标/题量/重点/难度）
  2. 老师确认后 → 拉学情 + 反向解析样卷 + 配比挑题 → 出卷面预览卡（22 道，分模块展示）
  3. 老师对话改细节（"压轴再难一点"/ "第 3 题换一道"）→ 局部更新卷面，不重新生成整卷
  4. 老师点"发布作业" → 写入二班作业本，学生端今晚 18:00 可见
**遇到问题怎么办**：
  - 题库返回不够（如条件下只有 18 道）→ 「这个范围只能找到 18 道，差 4 道；扩到相邻知识点凑齐？还是减题量？」
  - 学情数据拉不到 → 「没拿到二班最近的学情，按高考真题难度分布给你出，行不？」
  - 老师连续 3 轮不满意 → 主动提"要不换我的同事帮你？"（转人工 / 转传统组卷功能）
**说明的产品决策**：
  - 摘要预确认是核心交互（避免 AI 误解直接生成）
  - 局部修改用对话不用参数表单（B 端老师讨厌填字段）
  - 失败要主动给降级方案 + 问"行不行"，不直接报错
  - 不可逆动作（发布）必须显式确认
```

### 2.3 反模式（Agent 故事不该写的）

- ❌ 写 prompt 文案（"system: 你是教育助手..."）—— 算法侧的事
- ❌ 写技术实现（"调用 RAG 检索 + bge-m3 重排"）—— 研发自决
- ❌ 写 UI 细节（"按钮间距 16px / 动画 0.3s"）—— 交设计
- ❌ 写"AI 应该智能地..."这种空话 —— 智能体现在哪？写具体决策点
- ❌ 一条故事覆盖整个产品 —— 拆成多条，每条聚焦一个独立场景

### 2.4 GRFD 自检（Agent 故事质量门控）

每条 Agent 故事过 4 维：

| 维度 | 检查问题 | 不达标标准 |
|------|---------|-----------|
| **G**rounded（落地） | 触发场景够具体吗？ | 「用户说一句话」「在某个页面」太抽象 |
| **R**esponsive（响应） | AI 怎么做有 1-N 步可执行动作吗？ | 只有「智能处理」「自动回应」 |
| **F**ailsafe（兜底） | 写明失败 / 不满意时怎么办吗？ | 没写问题处理 / 失败时报错而非给降级方案 |
| **D**ecision（决策） | 能反推出至少 1 条产品决策吗？ | 只是流程描述，没有 PM 拍板的内容 |

自检后展示：
```
📋 Agent 故事质量检查（GRFD）

✅ AS-01 老师一句话组卷场景
   G✅ R✅ F✅ D✅

⚠️ AS-03 老师查学情场景
   G✅ R⚠️（"AI 自动分析"太抽象） F❌（没写数据为空时怎么办）D⚠️（没反推决策）

通过：{N} 条，待改进：{M} 条
```

- 全部通过 → 进入第三节
- 有不达标 → 询问继续 / 修改

## 三、Agent 工作流（仅 agent / hybrid 必写）

> **核心思路**：用 mermaid 状态机或泳道图把"AI 在每个状态做什么、何时切换、何时终止"画清楚。研发评审时一眼能看懂 Agent 在跑什么。
>
> **PM 视角**：状态 = 产品决策点；状态切换 = 用户行为或系统事件。**不画算法内部状态**（如"调用 LLM 中"、"等待 API 响应"——这些研发自决）。

### 3.1 写法

```
## WF-01 {主路径名}

\```mermaid
stateDiagram-v2
    [*] --> 待意图
    待意图 --> 摘要预确认: 老师说意图
    摘要预确认 --> 摘要预确认: 老师改摘要
    摘要预确认 --> 卷面预览: 老师确认
    卷面预览 --> 卷面预览: 老师改细节
    卷面预览 --> 已发布: 老师点发布
    卷面预览 --> [*]: 老师取消
    已发布 --> [*]
\```

**关键状态说明**：
- 待意图：欢迎屏，等老师输入
- 摘要预确认：AI 复述参数，等老师拍板
- 卷面预览：分模块展示题目，可对话改
- 已发布：写入作业本，跳转可见

**关键事件说明**：
- 老师改摘要：自然语言（"题量改 25"）触发，AI 重新生成摘要
- 老师改细节：自然语言（"压轴再难"）触发，AI 局部更新卷面
```

### 3.2 多场景拆多个工作流

如果产品有多个独立主路径（V1.1 搜题主路径 + V1.2 组卷主路径），分别画 WF-01、WF-02 等。

跨场景组的切换（题库 ↔ 数据应用）单独画一个工作流（WF-CTX），描述上下文如何切换 / 保留 / 重置。

### 3.3 自检（4 项硬规则）

| 维度 | 检查问题 | 不达标标准 |
|------|---------|-----------|
| 状态数适中 | 状态数在 3-7 范围吗？ | <3 太简化，>10 应拆子流程或简化 |
| 必有终态 | 有 `[*]` 终止节点吗？ | 没有，状态机不闭合 |
| 必有取消路径 | 用户能在中途退出吗？ | 没有"取消 / 返回 / 算了" 路径 |
| 关键状态有说明 | 每个状态有 ≤ 100 字解释吗？ | 状态名孤立，没说"这是什么、做什么" |

### 3.4 反模式

- ❌ 画算法内部状态（"调用 LLM 中"/"等待 RAG 召回"）—— 研发自决
- ❌ 画 UI 细节（"按钮 hover 状态"/"加载动画"）—— 交设计
- ❌ 一张图塞十几个状态 —— 拆子流程，主图保持 ≤ 7 状态
- ❌ 没有失败 / 取消路径 —— 健康的状态机必须可退出
