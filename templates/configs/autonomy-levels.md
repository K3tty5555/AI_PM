# Agent 自主性等级速查（L1–L5）

> 写 Agent / Hybrid 产品 PRD 时，必须给本期定档（参考 `agent-supplement.md` A3）。
> 借鉴自动驾驶分级法，描述本地化为中文。

---

## 一张图看懂

| 等级 | 一句话 | 谁拍板 | 谁执行 | 典型交互 |
|------|--------|--------|--------|---------|
| **L1** | 只给建议 | 人 | 人 | "这是 5 个候选，你自己挑" |
| **L2** | 替你做但你确认 | 人 | AI | "我组了一份，你点确认才入库" |
| **L3** | 默认替你做，可改可撤销 | AI（默认） | AI | "已自动按 6:3:1 分布，你可以拖动改" |
| **L4** | 异常时才打扰你 | AI | AI | 后台批量出卷，结果异常才推通知 |
| **L5** | 完全自动 | AI | AI | 定时任务，无人值守 |

## 详细定义

### L1 · 只给建议

- **AI 给方案，人决定一切**
- 适用：高风险场景、信任未建立、用户需要学习辅助判断
- 例：🩺 医疗诊断辅助 / 🎓 推荐题让老师挑 / 🛒 选品候选给运营 / 💬 给客服 3 个回复模板挑

### L2 · 替你做但你确认

- **AI 草拟，人点确认才生效**
- 适用：决策结果用户能快速判断对错、错了代价中等
- 例：🎓 组卷预览（→ 老师确认才入库）/ 💬 客服回复草稿（→ 客服点发送）/ 🛒 详情页文案（→ 运营点上架）/ 📄 邮件起草

### L3 · 默认替你做，可改可撤销

- **AI 直接做，用户随时调整**
- 适用：决策频次高、撤销机制成熟、错了能补救
- 例：🎓 默认 6:3:1 难度分布（可拖拽改）/ 💬 自动给工单打标签（可改标签）/ 📄 自动归类文档（可改分类）/ 💻 IDE Copilot inline suggestion

### L4 · 异常时才打扰你

- **AI 全自动，仅边界场景上报**
- 适用：流程稳定、有线上抽检、用户只需异常处理
- 例：🛒 自动审核详情页（仅合规风险词转人工）/ 💬 凌晨自动转值班（仅 SLA 风险升级）/ 📄 知识库定时整理 / 批量定时任务

### L5 · 完全自动

- **不需要人**
- 适用：纯后台 / 系统层 / 无用户感知
- 例：定时清理、统计聚合、模型微调任务

## 怎么定档（决策树）

```
1. 这个动作出错的代价大不大？
   ├─ 大（不可逆 / 涉及金钱 / 涉及合规） → L1 或 L2
   └─ 小（可撤销 / 影响有限）→ 继续往下

2. 用户能从结果一眼看出对错吗？
   ├─ 能 → L2 或 L3
   └─ 不能（要专业判断）→ L1

3. 用户是否需要每次确认？
   ├─ 需要 → L2
   └─ 不需要（只需偶尔关注）→ L3 或 L4

4. 用户是否需要全程在场？
   ├─ 需要 → L3
   └─ 不需要（事件驱动 / 定时）→ L4 或 L5
```

## 不同领域的默认起点

不同领域的容错率和撤销成本不同，起步等级也不同。以下是经验值，不是铁律。

### 高风险场景（教育 / 医疗 / 金融）：L2 起步

理由：
- 用户对 AI 决策的容错率低
- 错误传播链长（出错的题被学生练 → 错误认知固化；错诊导致后续治疗偏差）
- 老师 / 医生 / 客户经理需要"我做的"的可控感

### 中等风险场景（客服 / 电商运营 / 法务）：L2-L3 常见

理由：
- 决策结果用户能快速判断（回复对不对一眼能看出）
- 撤销成本中等（草稿可改、上架可下）
- 高频操作 L2 太累，L3 + 强撤销机制更适合

### 低风险场景（SaaS 协同 / 知识管理 / 个人助理）：L3 较多

理由：
- 错误代价小（分错文档可改）
- 撤销成本低（一键改回）
- 用户用工具高频，L2 每次确认体验断裂

### 多领域典型动作 → 等级映射

| 领域 | 典型动作 | 建议等级 | 理由 |
|------|---------|---------|------|
| 🎓 教育 | AI 推荐题给老师挑 | L1 | 老师挑题习惯强，AI 不替选 |
| 🎓 教育 | AI 组卷预览 | L2 | 必须老师点"发布"才落作业本 |
| 🎓 教育 | AI 默认难度配比 | L3 | 用户拖拽即可改 |
| 💬 客服 | AI 草拟回复 | L2 | 客服点发送才下发 |
| 💬 客服 | AI 自动给低优工单打标签 | L3 | 客服可改标签 |
| 💬 客服 | 凌晨自动转值班 | L4 | 仅 SLA 风险时上报 |
| 🛒 电商 | AI 选品候选 | L1 | 运营自己挑 |
| 🛒 电商 | AI 详情页文案 | L2 | 上架前必须人核 |
| 🛒 电商 | AI 默认排序/价格区间 | L3 | 运营拖拽即可改 |
| 📄 SaaS | AI 摘要会议 | L2 | 用户确认才入文档 |
| 📄 SaaS | AI 自动归类文档 | L3 | 用户随时改 |
| 📄 SaaS | 后台定时同步知识库 | L5 | 纯后台 |

## 写 PRD 时必填的三件事

在 A3.2 "本期定档" 一节，必须明确：

1. **本期目标等级**（L1-L5 中的一个或两个）
2. **MVP 先做到哪一档**（通常比目标低一档）
3. **为什么不是更高的等级**（容错率 / 信任建立 / 撤销机制 / 监管要求）

> 例 1（教育）：本期目标 L2-L3，MVP 先做 L2。理由：教育场景容错率低，先 L2 让老师建立信任，待试点 3 周采纳率 > 80% 后再升 L3，把"AI 默认难度分布"从摘要预确认下沉到默认接受。
>
> 例 2（客服）：本期目标 L3，MVP 先做 L2。理由：回复发送是高频操作，L2 每次确认体验差，但首版要先让坐席信任 AI 草稿质量，达到不修改直接发送率 > 40% 再升 L3。
>
> 例 3（电商）：本期目标 L2，明确不上 L3。理由：商品上架涉及资损风险（价格/合规词），即使采纳率高也不下放为默认接受，靠 L1 候选 + L2 详情页双档位即可。

## 反模式（要避免）

- ❌ 不写自主性等级 → 研发评审"这个 AI 会自动改用户数据吗"答不上
- ❌ 一开始就上 L4-L5 → 用户信任未建立，出错后没机会挽回
- ❌ 同一产品所有动作都同一等级 → 不同动作风险不同，要分别定档
- ❌ 等级和兜底机制不匹配（L3+ 没有撤销机制）→ 出错没法救
