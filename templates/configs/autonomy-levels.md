# Agent 自主性等级速查（L1–L5）

> 写 Agent / Hybrid 产品 PRD 时，必须给本期定档（参考 `agent-supplement.md` §A3）。
> 借鉴自动驾驶分级法，描述本地化为中文。

---

## 一张图看懂

| 等级 | 一句话 | 谁拍板 | 谁执行 | 典型交互 |
|------|--------|--------|--------|---------|
| **L1** | 只给建议 | 人 | 人 | "这是 5 个候选，你自己挑" |
| **L2** | 替你做但你确认 | 人 | AI | "我组了一份，你点确认才入库" |
| **L3** | 默认替你做，可改可撤销 | AI（默认） | AI | "已自动按 6:3:1 分布，你可以拖动改" |
| **L4** | 异常时才打扰你 | AI | AI | 后台批量出卷，结果异常才推通知 |
| **L5** | 完全自动 | AI | AI | 定时任务，无人值守 |

---

## 详细定义

### L1 · 只给建议

- **AI 给方案，人决定一切**
- 适用：高风险场景、信任未建立、用户需要学习辅助判断
- 例：医疗诊断建议、AI 推荐题让老师自己挑、AI 给三个文案让运营选

### L2 · 替你做但你确认

- **AI 草拟，人点确认才生效**
- 适用：决策结果用户能快速判断对错、错了代价中等
- 例：V1.2 组卷（AI 出卷 → 老师预览 → 点发布才落地）、AI 起草邮件等用户审

### L3 · 默认替你做，可改可撤销

- **AI 直接做，用户随时调整**
- 适用：决策频次高、撤销机制成熟、错了能补救
- 例：AI 默认按 6:3:1 难度分布（用户可拖动改）、IDE Copilot inline suggestion

### L4 · 异常时才打扰你

- **AI 全自动，仅边界场景上报**
- 适用：流程稳定、有线上抽检、用户只需异常处理
- 例：批量定时任务、自动审单（仅高风险转人工）、知识库定时更新

### L5 · 完全自动

- **不需要人**
- 适用：纯后台 / 系统层 / 无用户感知
- 例：定时清理、统计聚合、模型微调任务

## 怎么定档（决策树）

```
1. 这个动作出错的代价大不大？
   ├─ 大（不可逆 / 涉及金钱 / 涉及合规） → L1 或 L2
   └─ 小（可撤销 / 影响有限）→ 继续往下

2. 用户能从结果一眼看出对错吗？
   ├─ 能 → L2 或 L3
   └─ 不能（要专业判断）→ L1

3. 用户是否需要每次确认？
   ├─ 需要 → L2
   └─ 不需要（只需偶尔关注）→ L3 或 L4

4. 用户是否需要全程在场？
   ├─ 需要 → L3
   └─ 不需要（事件驱动 / 定时）→ L4 或 L5
```

---

## 教育场景默认建议

教育、医疗、金融场景普遍 L2 起步，**不轻易上 L3**。理由：
- 用户对 AI 决策的容错率低
- 错误传播链长（出错的题被学生练 → 错误认知固化）
- 老师 / 医生 / 客户经理需要"我做的"的可控感

| 智学网典型动作 | 建议等级 | 理由 |
|--------------|---------|------|
| AI 推荐题给老师挑 | L1 | 老师挑题习惯强，AI 不替选 |
| AI 组卷预览 | L2 | 必须老师点"发布"才落作业本 |
| AI 默认难度配比 | L3 | 用户拖拽即可改，撤销简单 |
| AI 自动剔除已考过的题 | L4 | 默认开启，设置项可关 |
| AI 定时同步学情数据 | L5 | 纯后台，无用户感知 |

## 写 PRD 时必填的三件事

在 §A3.2 "本期定档" 一节，必须明确：

1. **本期目标等级**（L1-L5 中的一个或两个）
2. **MVP 先做到哪一档**（通常比目标低一档）
3. **为什么不是更高的等级**（容错率 / 信任建立 / 撤销机制 / 监管要求）

> 例：本期目标 L2-L3，MVP 先做 L2。理由：教育场景容错率低，先 L2 让老师建立对 AI 的信任，待试点 3 周采纳率 > 80% 后再升 L3，把"AI 默认难度分布"从摘要预确认下沉到默认接受。

---

## 反模式（要避免）

- ❌ 不写自主性等级 → 研发评审"这个 AI 会自动改用户数据吗"答不上
- ❌ 一开始就上 L4-L5 → 用户信任未建立，出错后没机会挽回
- ❌ 同一产品所有动作都同一等级 → 不同动作风险不同，要分别定档
- ❌ 等级和兜底机制不匹配（L3+ 没有撤销机制）→ 出错没法救
