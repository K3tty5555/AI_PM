# 产品经理风格库

> 存放用户上传的产品经理风格配置，供 AI_PM 生成文档时使用

---

## 📁 文件夹结构

```
pm-styles/
│
├── README.md                          # 📄 本说明文件
│
├── default/                           # 🏭 系统默认风格
│   └── style-config.json
│
├── [用户风格1]/                       # 👤 用户上传的风格
│   ├── style-config.json              #    风格配置文件
│   └── examples/                      #    参考示例原文
│       └── original-prd.md
│
├── [用户风格2]/                       # 👤 另一个用户风格
│   ├── style-config.json
│   └── examples/
│       └── original-prd.md
│
└── ...
```

---

## 🚀 快速开始

### 1. 分析你的 PRD 创建风格

```bash
# 上传你的 PRD 到任意位置，然后执行
/ai-pm style analyze /path/to/your-prd.md
```

### 2. 查看已有风格

```bash
/ai-pm style list
```

### 3. 使用特定风格生成 PRD

```bash
# 在生成 PRD 时会提示选择风格
/ai-pm prd

# 或者直接指定风格
/ai-pm "需求描述" --style=你的风格名
```

## 📌 风格 vs 设计规范

| 对比项 | 产品经理风格 | 设计规范 |
|--------|-------------|---------|
| **关注点** | PRD 写作风格、结构、术语 | UI 视觉风格、组件、交互 |
| **存储位置** | `templates/pm-styles/` | `templates/design-systems/` |
| **影响内容** | 文字描述、章节结构、表格格式 | 原型视觉、颜色、字体、间距 |
| **使用时机** | 生成 PRD 文档时 | 生成原型时 |

---

## 💡 使用建议

1. **团队协作** → 整个团队使用统一风格，确保 PRD 一致性
2. **个人习惯** → 保存你的写作风格，后续生成更符合你的习惯
3. **不同产品类型** → B端/C端/数据产品可以有不同风格
4. **公司规范** → 上传公司要求的 PRD 模板作为标准风格

---

## 🔧 风格配置说明

`style-config.json` 包含以下维度：

| 配置项 | 说明 |
|--------|------|
| `structure` | 文档结构偏好（章节顺序、层级深度） |
| `writing` | 写作风格（语言特征、术语习惯） |
| `formatting` | 格式偏好（表格字段、标注方式） |
| `contentFocus` | 内容侧重（用户故事、数据指标比重） |
| `examples` | 风格示例片段 |

---

📁 **风格配置文件由 AI 自动生成，不建议手动修改**
