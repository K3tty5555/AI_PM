# 产品机会点用时间维度分层替代 P0/P1 编号

## 问题场景

在数据洞察报告和仪表盘中，用 P0/P1/P2/P3 来标注产品机会点优先级。但用户反馈：他们内部已经有 P0/P1 的优先级体系，含义不同，看到 P0 会误以为是最紧急需求，造成混乱。

## 解决方案

用**时间维度**替代数字编号：

| 旧写法 | 新写法 | 含义 |
|--------|--------|------|
| P0 | 紧迫 | 数据和技术条件都成熟，立刻可以推 |
| P1 | 推进 | 有基础，需补能力或协调资源后推 |
| P2/P3 | 打基础 | 依赖 AI/技术突破，先做数字化积累 |

仪表盘卡片分组示例：
```
── 紧迫 ──────────────────
[数学] 考试端接入全卷智批改；统一讲评入口
[物理] 同数学，合并推进

── 推进 ──────────────────
[英语] 作业-考试错题联动；讲评闭环落地
[化学] 考试端填空/客观题智批改；讲评率提升

── 打基础 ────────────────
[历史/地理/政治] 客观题自动批改；讲评流程标准化
```

## 验证数据

汉川一中报告和仪表盘均采用此分层，用户理解无歧义。

## 适用场景

任何含产品机会点分级的洞察报告或仪表盘，尤其当受众内部已有 P0/P1 优先级体系时。
