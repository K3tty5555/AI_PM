# 手动保存字段值优先于系统自动带入

## 问题场景
配置页面存在「系统自动带入」字段（如资源加工结果自动填充文体、作文题目），用户也可手动修改。当用户手动修改并保存后，下次进入页面或触发重新带入时，应使用哪个值？

## 解决方案
以用户最后一次手动保存的值为准，不覆盖。逻辑：用户已手动修改并保存 → 说明用户明确判断自动带入内容有误 → 系统应尊重用户的显式操作。

自动带入只在以下场景生效：
- 用户从未手动修改过（字段仍为系统默认值）
- 用户主动点击「重置为自动值」等明确操作

## 验证数据
ai辅批助手 M10 问题澄清：用户手动修改并保存后，再次进入页面保持手动值，不回填资源加工结果。

## 适用场景
- B端配置页，含有「自动带入 + 可手动覆盖」的字段
- 资源加工/模板/历史配置自动填充场景
- 适用于任何「系统推荐 vs 用户主动修改」冲突的表单设计
