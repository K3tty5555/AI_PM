# B端教育产品题型配置框架的文科/理科分类原则

## 问题场景

AI批改助手 V3 需要新增十余种学科题型配置页，若每个学科单独设计一套 UI 框架，工作量大且难以复用。

## 解决方案

按"是否需要公式编辑器"将所有学科题型归入两套框架：

**文科框架（纯文本）**
- 适用：语文主观题、政治/历史/地理/社会 填空/解答/简答
- 特征：题干/答案/解析均为纯文本输入框，无公式编辑器

**理科框架（含公式编辑器）**
- 适用：数学/物理/化学/生物/科学/信息技术 填空/解答/简答
- 特征：题干/答案/解析支持公式编辑器（LaTeX/MathML）

**附加规则**：
- 简答题与解答题产品侧不区分，配置页完全一致，题型标识由引擎侧区分
- 信息技术归理科框架（含编程语法，需公式/代码输入）
- 框架选择后，其余交互逻辑（批改进度、重置状态、开始批改判断）完全复用

## 验证数据

AI批改助手 V3 PRD 已按此框架设计，覆盖 10+ 学科题型，开发复用率估算 > 80%。

## 适用场景

B 端教育考试/作业产品中，涉及多学科题型配置的功能设计。
