# 教师数据查询的上下文自动绑定规则

## 业务规则
教师用自然语言查询学情数据时，系统须按以下规则自动收窄查询范围：

- **学科绑定**：当前页面学科自动限定查询范围（数学教师在数学报告页查"上次考试"，不会返回英语考试）
- **班级权限**：教师只能查看自己有授课关系的班级数据，跨班查询须先做权限验证
- **页面上下文决定数据维度**：在考试报告页 → 返回考试数据；在作业报告页 → 返回作业数据
- **同名学生必须追问**：出现同名学生时不能自动选一个，必须追问确认，错误的学生数据是严重问题

## 数据来源
教育超级智能体 PRD v3.2 数据查询规则（7.4 章）

## 为什么重要
不自动绑定上下文会让教师每次都要重复交代背景（学科/班级/时间），体验极差。更严重的是：如果不做权限收窄，教师可能查到无权限的班级数据；同名学生如果猜错，给到的是另一个学生的数据，是严重的数据准确性问题。

## 适用范围
所有 AI 对话式数据查询产品，尤其是 B 端多角色、多班级场景
