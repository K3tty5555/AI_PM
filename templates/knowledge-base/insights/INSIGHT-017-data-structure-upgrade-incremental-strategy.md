# 数据结构升级的增量改造策略

## 业务规则
当考试系统数据结构升级（如从大题粒度升至小题/多空粒度）时：

1. **历史数据不迁移**：存量考试继续按旧结构展示，不做全量转换
2. **新考试走新结构**：以考试创建时间为分界线，新建考试使用新数据结构
3. **UI 必须标识版本差异**：报告/导出页面需标注"该考试使用旧版制卡，按大题维度展示"
4. **下游全链路同步**：阅卷、扫描、工具箱、报告、导出模块必须同步支持新结构，否则新结构考试在某个环节会断链

## 数据来源
新版考试答题卡需求分析报告（MVP 边界 + 风险评估）

## 为什么重要
百万级历史考试数据迁移成本极高，且迁移过程中的任何错误都不可逆（影响学生历史成绩）。增量改造是唯一现实路径，但需要产品、研发、测试同步理解分支路由规则，否则容易出现"新卡在旧阅卷模块失败"的链路断裂问题。

## 适用范围
任何有大量历史数据的教育系统版本迭代，通用于任何 B 端系统的数据结构升级场景
