# "开始批改"≠"开始阅卷"——两个按钮的业务语义不同

## 结论

**"开始批改"** 是科组长触发 AI 引擎对全卷进行批改的动作，不是老师开始手动阅卷。
PRD v1.1 已将"开始阅卷"更正为"开始批改"，两者含义截然不同，设计时不可混淆。

## 两个动作的区别

| | 开始批改 | 开始阅卷 |
|--|---------|---------|
| **操作者** | 科组长 / 考试管理员 | 阅卷老师 |
| **入口位置** | 阅卷设置页，配置参数完成后 | 阅卷列表页 |
| **触发内容** | 向 AI 引擎提交批改任务，按试卷维度批量处理 | 老师逐份打开学生卷子手动评分 |
| **时序** | 考前/考后、阅卷开始前 | 批改完成（或批改中强制进入）后 |
| **完成标志** | 状态变"已完成"，显示完成时间 | 每份卷子提交最终分数 |

## 状态流转

```
配置参数 → [开始批改] → 批改中（X/Y份）→ 批改完成
                                              ↓
                                        [开始阅卷] → 老师逐份评分 → 提交
```

## 重新批改的含义

"重新批改"= 清除本轮 AI 批改结果，重新触发引擎；**配置参数不清空**。
触发前弹出二次确认：「重新批改将清除本轮AI批改结果，是否继续」。

## 为什么重要

原型或文案中若把两者混用，会造成：
- 科组长误以为点了按钮就开放了阅卷通道
- 老师误以为自己触发了 AI 批改
- 测试用例覆盖遗漏（两个按钮权限不同、时序不同）
