# 考试报告是事后复盘工具，不是阅卷中辅助工具

## 问题场景
在设计AI批改相关报告功能时，初稿将「作文分析报告」写成了阅卷过程中的侧边栏辅助工具，被用户否定并重写。

## 解决方案
考试报告（包含作文分析）的定位是：
- **触发时机**：考试结束、批改完成后，管理员主动点击「生成报告」
- **展示层级**：考试报告 → 班级报告 → 作文分析，支持下钻到单个学生
- **使用者**：教师/管理员用于事后复盘，不是阅卷过程中的辅助工具
- **生成方式**：整场考试统一生成，不单独触发作文报告

误解来源：「AI批改」容易让人联想到「批改时实时展示分析」，实际上报告是独立的复盘场景。

## 验证数据
ai辅批助手 6.4 章节重写：原稿（阅卷侧边栏）→ 修正后（考试报告-班级维度-作文分析，支持学生下钻）。

## 适用场景
- 考试类产品中涉及AI分析报告的功能定位
- 设计「AI分析」功能时，需明确区分「实时辅助」vs「事后复盘」两种场景
- PRD撰写：避免将报告功能混入阅卷流程主链路
