# 成绩查询的学期边界与延迟发布规则

## 业务规则
**学期边界**：
- 春季学期：2/1 — 7/15
- 秋季学期：9/1 — 次年 1/31
- 跨学期查询数据源完全不同，需明确用户意图

**延迟发布**：
- 学校常在考后延迟发布成绩（等排名确认、等校级审批），成绩从出炉到对教师可见可能有 1-3 天延迟
- 系统需支持分批发布（先对教师开放，再对家长开放）
- 成绩未发布期间，AI 查询端不得返回任何数据，包括"有多少学生参考"这类看似无害的信息

## 数据来源
教学监管 PRD + 教育超级智能体数据查询规则

## 为什么重要
用户说"上次考试"时，系统如果不理解学期边界，可能跨学期返回错误考试。延迟发布期间如果 AI 可以查到数据，相当于绕过了学校的发布控制，是严重的数据安全问题。

## 适用范围
所有含历史考试查询、跨时间范围数据分析的教育产品
