# 异步批改任务进度方案必须在 PRD 中明确

## 问题场景

PRD 写"展示进度条+百分比"，但没有定义数据从哪来、怎么来。前后端评审时双双 flag：轮询间隔多少？断线重连怎么处理？离页重进状态怎么恢复？这些是对接最先要确认的。

## 解决方案

进度条章节必须明确以下四项：
1. **获取方式**：WebSocket / SSE / 轮询（选其一）+ 间隔/协议
2. **进度定义**：已完成份数 / 总份数（口径对齐，避免前后端理解不同）
3. **断线重连**：网络断开后自动重连，不丢失进度
4. **离页恢复**：用户离开页面后重新进入，自动读取最新状态，不重置进度显示

## 验证数据

来自 ai辅批助手 评审报告 C3：前端和后端同时 flag，属于前后端对接最先需要确认的技术方案。

## 适用场景

所有带实时进度展示的异步任务 PRD（AI批改、导出、数据分析、批量操作等）。
