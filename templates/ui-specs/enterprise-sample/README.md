# 企业级B端产品设计规范（示例）

## 规范简介

- **规范名称**：Enterprise Standard
- **适用场景**：企业内部管理系统、B端产品
- **设计风格**：专业、稳重、高效
- **创建时间**：2026-02-28

## 核心特点

- **色彩**：深蓝主色，传递专业和信任
- **布局**：信息密度适中，注重效率
- **组件**：标准化组件，降低学习成本
- **响应式**：适配从笔记本到大屏显示器

## 设计原则

1. **效率优先**：减少操作步骤，提高工作效率
2. **信息清晰**：层次分明，重要信息突出
3. **一致性**：跨模块保持视觉和交互一致
4. **可访问性**：考虑色弱用户，不依赖颜色传递信息

## 使用说明

将此规范应用于企业内部系统开发，保持产品视觉统一。
