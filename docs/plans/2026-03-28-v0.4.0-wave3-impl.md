**Step 1: 创建 SKILL.md**

```yaml
---
name: ai-pm-pptx
description: >-
  PRD 转 PPT 演示文稿。读取 PRD Markdown，自动提取章节，生成幻灯片大纲，
  输出 .pptx 文件。支持 18 套配色方案和 4 种风格。
  当用户说「生成PPT」「导出PPT」「做个演示文稿」「PRD转PPT」时使用。
argument-hint: "[PRD路径]"
---
```

三层编排：
1. 规划层：PRD → 章节提取 → 幻灯片大纲（每页类型+标题+要点）
2. 设计层：行业预设 → 配色+风格+字体打包推荐
3. 生成层：调用 `prd2pptx.py` 编译 .pptx

命令路由：
- `/ai-pm pptx` — 从当前项目 PRD 生成
- `/ai-pm pptx [路径]` — 从指定 PRD 文件生成

**Step 2: 创建 design-system.md**

18 套配色方案，按分组：
- 推荐（根据行业匹配）
- 商务：business-authority / platinum-white-gold / tech-blue
- 创意：dreamy-creative / bohemian / art-food / coastal-coral
- 自然：nature-outdoor / forest-eco
- 学术：vintage-academic / education-chart
- 科技：tech-vibrant / tech-nightscape
- 其他：modern-health / artisan-handmade / elegant-fashion / luxe-mystery / orange-mint

4 种风格配方 + 字体搭配规则。

**Step 3: 创建 page-types.md**

5 种页面类型：
- Cover — 封面页
- Section — 章节分隔页
- Content — 内容页（3 种子布局：bullet-list / two-column / image-text）
- Chart — 数据图表页
- End — 结束页

布局多样性规则：相邻 Content 页不同子布局。

**Step 4: 创建 pitfalls.md**

- python-pptx 的 Inches/Pt/Emu 单位换算
- 标题下方不用强调线（用留白替代）
- 中文字体必须用完整字体名
- 母版布局引用需精确匹配 slide_layout.name

**Step 5: 创建 qa-workflow.md**

QA 流水线：生成 → markitdown 提取文本 → 占位符检测 → 内容完整性校验。

**Step 6: 创建 prd2pptx.py**

核心 Python 脚本（~400 行），接收参数：

```bash
python3 prd2pptx.py \
  --input <prd.md> \
  --output <output.pptx> \
  --outline <outline.json> \
  --color-scheme <scheme-name> \
  --style <style-name> \
  --font-zh <中文字体> \
  --font-en <英文字体>
```

功能：
- 解析 outline.json（LLM 生成的大纲）
- 按页面类型创建幻灯片
- 应用配色方案和字体
- 输出 .pptx

**Step 7: 创建 verify.py**

```bash
python3 verify.py <output.pptx>
```

- 尝试用 markitdown 提取文本（需安装）
- fallback：用 python-pptx 直接提取
- 检查占位符残留
- 输出 JSON 报告

```bash
git add .claude/skills/ai-pm-pptx/
git commit -m "feat(skill): add ai-pm-pptx — PRD to PPT with 18 color schemes, 5 page types"
```

```markdown
### 排版原则审查（设计师专属）

对原型 HTML 逐项检查以下六条排版原则，每条给出 ✅/⚠️/❌ + 问题描述 + 修复建议：

5. **重复与一致** — 同级元素是否使用相同样式？是否有孤立格式（仅一处加粗、仅一处用不同色）？
6. **视觉层次** — 用户视线路径是否清晰？关键操作按钮是否突出？
```
