## 1. 产品定位

面向产品经理个人用户的 Web 应用，将现有 Claude Code 版 AI PM 的能力以可视化、阶段引导式的交互呈现出来。

**核心理念**：SKILL.md 是唯一的业务逻辑源，Web 应用是展示层。Claude Code 里改了技能，Web 端自动生效。

**与现有 AI PM 的关系**：完全共存，互不影响。

```
用户电脑
├── AI_PM/                  ← 现有项目，继续在 Claude Code 里用
│   ├── .claude/skills/     ← 技能文件（两边共享读取）
│   ├── templates/          ← 模板（两边共享读取）
│   └── output/projects/    ← 产出文件（两边共享读写）
│
└── ai-pm-web/              ← 新项目，独立的 Web 应用
    └── （自己的代码）
```

## 2. 技术选型

| 层 | 选择 | 理由 |
|---|------|------|
| 前端 | Next.js 15 (App Router) + Tailwind + shadcn/ui | 全栈一体，组件库成熟 |
| LLM | Anthropic SDK → Claude API | 直接调，不经 Claude Code |
| 数据库 | SQLite (via Drizzle ORM) | 零运维，单用户足够 |
| 文件存储 | 本地磁盘 `output/projects/` | 与 Claude Code 版共享同一目录 |
| 实时通信 | Server-Sent Events (SSE) | Claude API 流式输出 → 前端实时渲染 |
| 后续桌面化 | Tauri | 比 Electron 更轻量 |

### API 配置

自动读取用户本地 Claude Code 的环境配置（API Key / 代理地址 / 认证凭证），零配置启动。仅在读取失败时弹出手动设置页面作为兜底。

设置页面支持：
- 认证密钥（不限格式，兼容代理服务的任意凭证）
- 自定义 API 地址（兼容代理 / 中转服务）
- 模型选择

## 3. 核心架构

### skill-loader 机制

```
用户触发"开始分析"
  → skill-loader 读取 ai-pm-analyze/SKILL.md
  → 拼接 system prompt：SKILL.md + 项目上下文 + 用户输入
  → 调 Claude API (streaming)
  → SSE 推送到前端实时渲染
```

### 模块结构

```
ai-pm-web/
├── app/                    # Next.js App Router
│   ├── (dashboard)/        # 项目列表、全局看板

│   │   ├── requirement/    # 阶段1：需求输入
│   │   ├── analysis/       # 阶段2：需求分析
│   │   ├── research/       # 阶段3：竞品研究
```

## 4. 设计风格：终末地

延续 AI_PM 教程中心的设计语言。

### Design Tokens

```
颜色：
  --white: #FFFFFF
  --light: #F5F5F5
  --dark: #191919
  --dark2: #252525
  --yellow: #fffa00          ← 主强调色，仅用于可交互元素和进度
  --green: #4CAF50
  --teal: #4ECDC4
  --text: #141414
  --text-muted: #6b6b6b
  --border: rgba(0,0,0,0.10)

字体：
  --font-mono: 'Courier New', Consolas, monospace     ← 标签、编号、HUD元素
  --font-sans: -apple-system, PingFang SC, sans-serif  ← 正文、标题

缓动/过渡：
  --ease-out: cubic-bezier(0.16, 1, 0.3, 1)
  --tr: 0.28s cubic-bezier(0.16, 1, 0.3, 1)

圆角：0px（全局直角，仅状态 dot 例外）
阴影：0 2px 16px rgba(0,0,0,0.04)，hover 时黄色辉光
间距：8px 基准网格
```

### 视觉要素

| 要素 | 说明 |
|------|------|
| `//` 前缀 | 品牌符号，用于顶栏和标题 |
| 六边形节点 | 阶段导航，已完成=黄色填充，当前=黄色边框+脉冲，未解锁=灰色 |
| 左侧竖条 | 卡片重要性标记：金色=P0/需操作，青色=P1/建议，灰色=P2/补充 |
| 绿色脉冲点 | 系统在线状态指示 |
| HUD 角标 | monospace + letter-spacing 的区块标签（如 `ANALYSIS_RESULT`） |
| 黄色进度条 | `linear-gradient(90deg, #fffa00, rgba(255,250,0,0.3))` |

## 5. 页面交互设计

### 5.1 整体布局

```
┌──────────────────────────────────────────────────────┐
│  // AI PM                            ● online  [⚙]  │  ← 顶栏
├────────────┬─────────────────────────────────────────┤
│  项目列表   │  ⬡━━⬡━━⬡──⬡──⬡──⬡──⬡  阶段导航       │
│            ├─────────────────────────────────────────┤
│            │                                         │
│            │           主工作区                       │
│            │                                         │
│            │                                         │
├────────────┴─────────────────────────────────────────┤
└──────────────────────────────────────────────────────┘
```

### 5.2 项目仪表盘

- 项目列表卡片：左侧金色竖条=进行中，灰色=已完成
- 每张卡片显示：项目名、最后更新时间、当前阶段、进度条
- 右上角 [+ NEW] 按钮新建项目

### 5.3 需求输入（阶段1）

- 项目名称输入
- 富文本编辑器（支持粘贴文档内容）
- 文件上传区（拖入 PDF/Word/Excel/图片）
- [开始分析 →] 按钮进入下一阶段

### 5.4 需求分析（阶段2）

- 分析结果以卡片展示：目标用户、核心痛点、功能范围
- 功能范围用横向条形图表达优先级权重
- AI 追问以内联卡片形式出现（非弹窗），提供快捷选项 + 自由输入
- 回答后气泡折叠为一行摘要
- 确认后进入下一阶段

### 5.5 竞品研究（阶段3）

- 竞品对比卡片（并排展示，维度评分）
- 差异化机会高亮提取
- （第二期实现）

### 5.6 用户故事（阶段4）

- 按优先级分组（P0/P1/P2），每组可折叠
- 故事卡片可展开编辑验收标准
- 支持拖拽排序调整优先级
- 可手动添加故事

### 5.7 PRD（阶段5）

- 左侧：PRD 富文本预览，支持点击段落直接编辑
- 右侧：目录导航，点击跳转
- Mermaid 流程图实时渲染
- 底部 AI 辅助输入框（内联式）
- 导出按钮组：PDF / 飞书 / Markdown
- （MVP 仅支持预览和编辑，导出第二期）

### 5.8 原型预览（阶段6）

- iframe 内嵌原型 HTML，直接可交互
- 设备尺寸切换：📱移动端 / 🖥 Web端
- 页面列表切换
- （第二期实现）

### 5.9 六角色评审（阶段7）

- 技术组（前端/后端/测试）+ 产品组（数据/交互/体验）
- 每个角色一张意见卡片，按严重程度标记
- 评审摘要：总意见数、需处理数、已通过数
- [采纳修改并更新 PRD] 一键回写
- （第二期实现）

### 5.10 状态设计

| 状态 | 表现 |
|------|------|
| 空状态 | 六边形图标 + 一行引导文案 |
| 生成中 | 黄色渐变进度条，1.6s 动画 |
| 生成失败 | 红色左侧竖条卡片 + 报错信息 + 重试按钮 |

### 5.11 交互原则

- **阶段引导为主框架**：每一步有明确的输入、输出和下一步指引
- **关键节点保留轻量对话**：AI 追问以内联气泡形式融入页面，不是独立聊天窗口
- **渐进披露**：卡片默认收起，点击展开详情，避免信息过载

## 6. 分期计划

### MVP（第一期）：核心链路

**范围**：仪表盘 → 需求输入 → 需求分析 → 用户故事 → PRD 生成/预览/编辑

**不含**：竞品研究、原型生成、评审、PRD 导出、桌面打包

**里程碑**：

```
M1  项目脚手架 + 设计系统
    Next.js 初始化、终末地 design tokens、基础组件库
    （按钮、卡片、六边形导航、内联对话、进度条）

M2  后端核心
    skill-loader、Claude API 封装（SSE 流式）、
    读取本地 Claude 配置、SQLite 项目存储

M3  仪表盘 + 需求输入页
    项目 CRUD、富文本编辑器、文件上传

M4  需求分析 + 用户故事页
    AI 流式输出 → 卡片渲染、内联追问交互、
    故事拖拽排序

M5  PRD 生成 + 预览编辑
    Markdown 渲染、Mermaid 流程图、
    点击即编辑、目录导航

M6  联调打磨
    全链路走通、动画打磨、空状态/错误态、响应式适配
```

### 第二期：补齐全流程

- 竞品研究页面
- 原型生成 + iframe 预览
- 六角色评审
- PRD 导出（PDF / 飞书 / Word）
- 项目归档与搜索

### 第三期：超越 CLI 版本

- PRD 版本对比（逐段 diff）
- 评审意见追踪（采纳/驳回/理由）
- 项目数据看板（跨项目统计）
- 模板市场
- Tauri 桌面打包
