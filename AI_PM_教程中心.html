产品流程类

7 个技能
🎯
AI_PM 主控
/ai-pm

AI 产品经理。输入需求，引导完善，输出 PRD + 原型。支持需求分析、竞品研究、用户故事、PRD、原型、评审全流程。多项目管理，断点续传。复杂需求自动启用多代理协作（/ai-pm --team）。

🔍
需求分析
/ai-pm-analyze

深入分析产品需求，挖掘用户画像、核心痛点、功能范围和优先级。通过对话引导用户补充信息，不满足于表面描述。

🔭
竞品研究
/ai-pm-research

分析市场同类产品，输出功能对比、差异化机会和市场定位建议。

📖
用户故事
/ai-pm-story

基于需求分析和竞品研究，编写详细的用户故事和验收标准。

📄
PRD 生成
/ai-pm-prd

整合需求分析、竞品研究、用户故事，输出完整的产品需求文档。支持产品分身写作风格和设计规范。

🖥️
原型生成
/ai-pm-prototype

基于 PRD 生成可交互的单页网页原型，支持移动端和 Web 端。自动应用产品设计规范，默认遵循 Apple HIG。

⚖️
需求评审
/ai-pm-review

模拟真实评审会议，九大专家角色从专业角度评审 PRD 和原型，输出评审意见和修改建议。支持多轮迭代。

使用指南

从零开始，10 分钟上手

跟着步骤走，第一次就能产出完整 PRD。

第一次使用

1
安装 Claude Code
前往 Anthropic 官网下载并安装 Claude Code CLI，需要 Claude Pro 或 API 账号。
2
克隆 AI_PM 项目
将 AI_PM 项目克隆到本地，并在 Claude Code 中打开该目录。技能文件会自动被识别。
3
输入 /ai-pm 查看状态
首次运行会显示欢迎界面，告诉你如何开始第一个项目。无项目时会看到引导提示。
4
输入需求开始第一个项目
直接描述你的产品需求，AI 会引导你逐步完善，并自动执行分析、研究、PRD、原型全流程。

常用工作流

完整新产品
/ai-pm "你的需求描述" → 全自动执行到原型
从需求澄清开始，经过分析、竞研、用户故事、PRD，最终生成可交互原型。
快速周报
/ai-pm weekly → 输入本周工作 → 生成周报
随意描述这周干了什么，AI 自动归类整理，生成向上汇报版或团队同步版周报。
需求评估
/ai-pm priority → 粘贴需求列表 → 获得优先级排序
支持 Excel/CSV 文件或直接粘贴需求，四维评分模型自动排序，生成可发送的回复模板。
现场调研
/ai-pm interview → 访谈 → 自动生成 PRD
带着客户在场时使用，边聊边记录，访谈结束立刻产出需求草稿，当场确认。

效率技巧

输入「加急」或「yolo」跳过所有确认
在 /ai-pm 主流程中输入「加急」，AI 会自动执行 Phase 1-4，仅在 PRD 生成前停一次确认，生成原型前再停一次，其余全部自动完成。
用 --team 启动多代理，处理大型复杂需求
输入 /ai-pm --team "需求描述" 或直接 /agent-team "需求描述"，多个 AI 代理并行工作，速度比标准模式快约 40%。适合需求文字超过 200 字、包含多个独立功能模块的复杂项目。
配置产品分身，PRD 风格越来越像你
用 /ai-pm persona analyze 分析你历史写的 PRD，AI 提取你的写作特征（用词、句式、结构偏好），之后生成的所有 PRD 都会模仿你的风格。分析文档越多越准确。

工作流模式

四种模式，按需选择

根据项目复杂度和时间约束，选择最合适的执行路径。

1
标准流程
最完整，质量最高，适合新产品立项
需求澄清 → 需求分析 + 竞品研究 → 用户故事 → PRD → 原型 → 九角色评审

Phase 2+3 并行执行（需求分析与竞品研究同时进行），PRD 生成前有一次统一确认节点，原型生成前有 Token 消耗提示。整个流程覆盖产品生命周期所有关键输出物。

/ai-pm "我想做一个帮用户决定吃什么的 App"
2
快速模式
跳过中间阶段，直达 PRD 和原型
需求描述 → ⚡ 加急 → 自动分析 → PRD 确认 → 原型生成

在主流程中输入「加急」或「yolo」，Phase 1-4 全自动执行，仅在 PRD 生成前停一次确认，生成原型前再停一次。适合已经想清楚需求、只需要快速产出文档的场景。

/ai-pm "需求描述" → 输入 "加急"
3
多代理模式
并行执行，速度快 40%，适合大型复杂需求
Wave 1: PM + 分析师 ↓ Wave 2: PRD 专家 ↓ Wave 3: 评审专家

多个 AI 代理并行执行独立子任务。Wave 1 产品经理和竞品分析师同时工作，Wave 2 读取 Wave 1 输出生成 PRD，Wave 3 执行三角色精简评审。需求超过 200 字或包含多个独立模块时建议使用。

/ai-pm --team "复杂需求描述"
4
现场调研模式
有客户在场时使用，边访谈边产出
准备提问 → 逐步访谈 → 实时记录 → 需求草稿 → 客户确认 → 正式 PRD

专为线下拜访客户场景设计。AI 生成提问方向，你逐步访谈，AI 同步记录用户原话、痛点和期望。访谈结束立刻生成需求草稿，客户当场确认，然后触发 PRD 生成。生成内容要快，客户不能等超过 30 秒。
