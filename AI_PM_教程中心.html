AI PM
GETTING STARTED

快速开始

两种使用方式，根据你的偏好选择。Claude Code 命令行原生集成，桌面客户端可视化操作。

01
克隆项目

把项目拉到本地。没用过命令行？把下面这行复制到终端回车即可。

$ git clone https://github.com/K3tty5555/AI_PM.git
02
打开目录

用 Claude Code 打开 AI_PM 目录（File → Open Folder）。还没装？前往 claude.ai/code。

> File → Open Folder → AI_PM/
03
输入需求

用 /ai-pm 命令描述你的想法。AI 会先确认方向，然后自动推进到 PRD 和原型。

/ ai-pm "帮年轻人记账的小程序"
01
下载安装

从 GitHub Releases 下载对应平台的安装包，双击安装。

→ github.com/K3tty5555/AI_PM/releases
02
配置 AI 后端

设置中选择：Anthropic API（填 Key）/ OpenAI 兼容接口 / Claude CLI（复用本机登录）。

~ 设置 → AI 后端 → 选择模式
03
创建项目

点击「新建项目」，输入需求。每个阶段支持「先聊聊」模式——和 AI 讨论达成共识后再生成。

+ 新建项目 → 输入需求 → 开始
SKILLS

技能列表

21 个项目技能，覆盖完整产品流程、扩展工具与配置管理。命令直接复制到 Claude Code 输入框运行。

/ai-pm 核心
AI PRODUCT MANAGER

不知道从哪里开始就从这里开始。告诉 AI 你想做什么，它会一步步引导你把需求、竞品、PRD、原型都跑完。中途可以随时暂停，下次接着来。

/ai-pm "帮年轻人记账的小程序"
/ai-pm-analyze 核心
REQUIREMENTS ANALYSIS

把模糊的需求问清楚。会问你目标用户是谁、痛点在哪、想做哪些功能，整理成分析报告存下来。产出：02-analysis-report.md

/ai-pm-analyze "企业内部审批系统"
/ai-pm-research 核心
COMPETITIVE RESEARCH

扫一遍市场上的同类产品，看他们做了什么、没做什么，帮你找到差异化的切入点。产出：03-competitor-report.md

/ai-pm-research "记账类 App"
/ai-pm-story 核心
USER STORY WRITER

把功能拆成用户故事格式，每条附带验收标准，可以直接给开发团队用。产出：04-user-stories.md

/ai-pm-story
/ai-pm-prd 核心
PRD GENERATOR

把前面分析的结论整合成一份完整 PRD。默认输出 Markdown，按需导出 DOCX/PDF/分享页；客户端可评分、对比版本、让 AI 插图。产出：05-prd/05-PRD-v1.0.md

/ai-pm-prd --export=pdf
/ai-pm-prototype 核心
PROTOTYPE GENERATOR

根据 PRD 生成一个能点击的 HTML 原型。可读取设计规范或既有代码的设计指纹，生成后做完整性审计。双击 index.html 直接预览。

/ai-pm-prototype
/ai-pm-review 核心
REQUIREMENTS REVIEW

模拟一场评审会。产品、设计、前端、后端、测试、运营——六个角色同时开口，从各自立场找问题、给意见。产出：08-review-report-v1.md

/ai-pm-review --round=2
/ai-pm-driver 核心
PM QUALITY GATE

评审前的产品经理守门。按 KettyWu 风格检查 PRD 的目标、边界、指标、表达和可落地性，先把硬伤挡在评审会前。

/ai-pm-driver 05-prd/05-PRD-v1.0.md
/ai-pm-priority 扩展
PRIORITY EVALUATOR

把各方提来的需求集中评估。按业务价值、实现难度、用户影响、战略方向四个维度打分，排出优先级，顺带生成一份可以直接发给提报人的回复。

/ai-pm-priority
/ai-pm-weekly 扩展
WEEKLY REPORT

把这周做的事随便说一遍，AI 帮你归类整理成周报。可以出两个版本：给上级看的简洁版，和给团队看的详细版。

/ai-pm-weekly
/ai-pm-interview 扩展
FIELD INTERVIEW

带着电脑去和客户面谈时用。AI 给出访谈提纲，你跟着走，谈完直接生成需求草稿，不用事后再整理。

/ai-pm-interview
/ai-pm-data 扩展
DATA INSIGHT

上传你的业务数据表，AI 读出规律，生成一个浏览器里能点的交互图表页面，并告诉你数据背后可能藏着哪些产品机会。

/ai-pm-data 用户行为数据.xlsx
/ai-pm --team 扩展
MULTI-AGENT MODE

复杂需求用多代理并行处理。产品分析师拆需求、竞品研究员做市场分析、主控 PM 整合产出，比标准模式快约 40%。

/ai-pm --team "企业级 CRM 系统"
工具广场 / PPT 扩展
PPT GENERATOR

客户端功能广场里的 PPT 生成能力。从 PRD 或大纲生成演示文稿，配色和页面类型自动编排，大纲可调整后一键输出 .pptx 文件。

客户端：功能广场 → PPT 生成

GAP RESEARCH

围绕机会缺口做研究，不只罗列竞品，而是找目标用户、供给能力、体验断点和可切入空间之间的差距。

/pm-gap-research "AI 教学助手市场机会"

MULTI-PERSPECTIVE REVIEW

对方案、文档或页面做多视角审查。适合在正式需求评审之外，快速补齐业务、体验、技术和风险视角。

/multi-perspective-review "评审这份方案"

/pdf-skill "把这份 PRD 导出为 PDF"
/ai-pm-persona 配置
PRODUCT PERSONA
