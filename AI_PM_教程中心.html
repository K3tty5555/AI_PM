//GETTING STARTED

快速开始

01
TERMINAL
克隆项目

打开终端，把项目拉到本地。没用过命令行？把下面这行复制进去回车就行。

$ git clone https://github.com/K3tty5555/AI_PM.git && cd AI_PM
02
CLAUDE CODE
打开目录

Claude Code 是 Anthropic 的官方命令行工具。装好后用它打开 AI_PM 目录（File > Open Folder）。还没装？→ claude.ai/code

> File > Open Folder > AI_PM/
03
SKILL
输入需求

输入命令描述你的想法，AI 会先问几个问题确认方向，然后自动推进。产出文件在 output/projects/ 目录下。

/ ai-pm "帮年轻人记账的小程序"

前置条件：已安装 Claude Code · 产出文件在 output/projects/{项目名}/ · 支持断点续传 · 多项目隔离

技能列表

悬停卡片查看示例 — 点击「复制命令」粘贴到 Claude Code 运行

/ai-pm P0 ◆◆◆
AI PRODUCT MANAGER

不知道从哪里开始就从这里开始。告诉 AI 你想做什么，它会一步步引导你把需求、竞品、PRD、原型都跑完。中途可以随时暂停，下次接着来。

// USAGE
/ai-pm "帮年轻人记账的小程序"
加急 "帮年轻人记账的小程序" （跳过追问，直跑到原型）
/ai-pm --team "企业级 CRM 系统"
/ai-pm continue
/ai-pm list
CORE
/ai-pm-analyze P0 ◆◆◆
REQUIREMENTS ANALYSIS

把模糊的需求问清楚。会问你目标用户是谁、痛点在哪、想做哪些功能，整理成分析报告存下来。产出：02-analysis-report.md

// USAGE
/ai-pm-analyze "企业内部审批系统"
/ai-pm-analyze "医疗问诊 App"
CORE
/ai-pm-research P0 ◆◆◆
COMPETITIVE RESEARCH

扫一遍市场上的同类产品，看他们做了什么、没做什么，帮你找到差异化的切入点。产出：03-competitor-report.md

// USAGE
/ai-pm-research "记账类 App"
/ai-pm-research "企业 OA 系统"
CORE
/ai-pm-story P0 ◆◆◆
USER STORY WRITER

把功能拆成用户故事格式，每条附带验收标准，可以直接给开发团队用。产出：04-user-stories.md

// USAGE
/ai-pm-story
CORE
/ai-pm-prd P0 ◆◆◆
PRD GENERATOR

把前面分析的结论整合成一份完整 PRD。DOCX 格式可以直接导入飞书变成可编辑云文档。产出：05-prd/05-PRD-v1.0.md

// USAGE
/ai-pm-prd
/ai-pm-prd --export=pdf
/ai-pm-prd --export=docx （导飞书用这个）
/ai-pm-prd --export=all
CORE
/ai-pm-prototype P0 ◆◆◆
PROTOTYPE GENERATOR

根据 PRD 生成一个能点击的 HTML 原型。首次使用时选设计规范（公司规范 / AI 情境定制 / 主流组件库），项目内记住偏好。双击 index.html 直接预览。

// USAGE
/ai-pm-prototype
CORE
/ai-pm-review P0 ◆◆◆
REQUIREMENTS REVIEW

模拟一场评审会。产品、设计、前端、后端、测试、运营——六个角色同时开口，从各自立场找问题、给意见。产出：08-review-report-v1.md

// USAGE
/ai-pm-review
/ai-pm-review --round=2 （第二轮修改后再评）
CORE
/ai-pm-priority P1 ◆◆
PRIORITY EVALUATOR

把各方提来的需求集中评估。按业务价值、实现难度、用户影响、战略方向四个维度打分，排出优先级，顺带生成一份可以直接发给提报人的回复。

// USAGE
/ai-pm-priority

把这周做的事随便说一遍，AI 帮你归类整理成周报。可以出两个版本：给上级看的简洁版，和给团队看的详细版。

// USAGE
/ai-pm-weekly
